## 1. 何をつくるか

### 成果物

ブラウザで動作する Web アプリケーション（静的サイト）。営業担当者が見積作成時に、商品の EC 販売価格・仕入値・粗利率を一覧で確認するためのツール。

- ホスティング費・保守運用費は発生しない設計（Vercel 等の無料枠を想定）
- ユーザー認証なし（社内利用のみ）
- トラックス見積書システムとの連携はしない（本ツールは見積作成の参考情報を提供する位置づけ）

### 機能一覧

| # | 機能 | 概要 | ステータス |
|---|---|---|---|
| F1 | CSV 取込・データ統合 | メッシュ・ネトロン・トリカルの在庫 CSV をアップロードして取り込み | 実装済み |
| F2 | 商品検索・販売価格算出 | 規格条件で検索し、仕入値・固定費・粗利率から販売価格を算出して表示 | 要改修（後述） |
| F4 | 代替提案 | 在庫 0 の商品に対し、仕様が近い代替商品を提案 | 実装済み |

## 3. どういうロジックでアウトプットをつくるか

### 3-1. 販売価格の算出

#### 採用する計算式

クレバー社回答の計算式を採用する:

```
販売価格 = (仕入単価 × カット長 + 固定費) / (1 - 粗利率)
```

※ カット長が計算に必要かどうかは要確認 D で確定後。仕入単価にカット長が織り込み済みの場合、計算式は `販売価格 = (仕入単価 + 固定費) / (1 - 粗利率)` に簡略化される可能性がある。

#### ツールでの使い方（MTG 2026-06-09 で方向転換）

本ツールでは仕入値・固定費・粗利率から販売価格を**算出する**。従来の「EC サイトから販売価格を取得して粗利率を逆算する」方針から転換。

くればぁ側の実態: 仕入れ値変更時に計算式で販売価格を算出 → EC サイトに手動反映。販売価格のデータベースは存在しない。本ツールがそのデータベースの役割を担う。

入力:
- **仕入単価**: 在庫 CSV から取得（要確認 B でカラム確定後）。仕入れ値変更時はツール画面で手動更新
- **カット長**: 要確認 D で確定後
- **粗利率**: 全商品共通のデフォルト値を設定（例: 50%）。編集可能だが、基本は固定費で調整する運用
- **固定費**: デフォルト 6,000 円。**商品ごとに編集可能**（送料等の違いを吸収するための調整変数）

出力:
- **販売価格（円）**: 商品ごとの EC 販売価格

#### 固定費の運用（MTG 2026-06-09 合意）

固定費は主に送料を反映する。商品の仕入れ先・サイズ（幅）によって送料が大きく異なるため、商品ごとに調整できる必要がある。

- 大半の商品: デフォルト値（6,000 円）のまま一括適用
- 送料が大きい商品（大型・遠方仕入れ先等）: 個別に固定費を編集
- 一括入力と個別入力の両方ができる UI が必要
- 固定費を個別調整する商品のフィルタ条件: 仕入れ先、幅（要確認 F）

#### 仕入れ値変更の運用（MTG 2026-06-09 確認）

- 仕入れ値変更の自動検知の仕組みはない（完全にアナログ）
- ツールの画面で手動入力する運用で問題ない（くればぁ了承）
- 仕入れ値を入力したら、計算式で販売価格が自動更新されるのが理想（くればぁ合意）

#### 在庫表との連携（MTG 2026-06-09 確認）

在庫表と価格データの連携は nice-to-have。くればぁ代表:「最悪在庫表の連携が難しければ別々でも大丈夫」。

#### 現行実装との差分

現行実装（price-calculator.ts）は `仕入値 × 1.5 × 1.1` で税込単価を算出している。これは上記のロジックと異なるため改修が必要。

※ 最終確定はフロー図レビュー後（6/18 MTG）。方向性は MTG 2026-06-09 で合意済み。

### 3-2. 代替提案

| 項目 | 内容 |
|---|---|
| トリガー | 検索した商品の在庫が 0 の場合 |
| 提案条件 | 目開きが同じ、またはメッシュ数が同じで、在庫 > 0 の商品 |
| 表示内容 | 品番、幅、カラー、残り(m)、EC 販売価格、粗利率 |

根拠: クレバー社回答「基本的には目開きまたはメッシュ数が同じ、近いもの」

「近い」の定義: クレバー社の回答では「同じ、近いもの」とされている。まずは「同じ値」を基本条件とし、該当がない場合に範囲を広げる方針で実装する。

#### [要確認 E] ネトロン・トリカルでの代替提案

ネトロン・トリカルの CSV には「目開き」「メッシュ数」のカラムが存在しない。これらのカテゴリでの代替提案の判定基準を確認する必要がある。

※ 中林が確認予定。回答がなければユーザーが情報を提供。
